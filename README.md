ERP & POS System (Backend)

Серверна частина (REST API) для комплексної веб-орієнтованої ERP та POS системи. Побудована на базі Django та Django REST Framework. Забезпечує бізнес-логіку для управління касами, складським обліком, ремонтами, фінансами та генерацією звітності.

## Технологічний стек

- Мова: Python 3.10+
- Фреймворк: Django 6.0.5, Django REST Framework (DRF)
- База даних: PostgreSQL (локально підтримується SQLite)
- Автентифікація: JWT (JSON Web Tokens) через rest_framework_simplejwt
- Зберігання медіа: Cloudinary (зображення товарів та фотографії ремонтів)
- Генерація PDF: ReportLab (фіскальні чеки, квитанції ремонтів з кириличними шрифтами)
- Контейнеризація: Docker та Docker Compose
- Документація API: Swagger UI / Redoc (через drf-spectacular)

## Основні модулі системи (Django Apps)

- `users` (Користувачі та Авторизація): Кастомна модель користувача (вхід за Email). Розмежування прав доступу на основі ролей (Admin, Seller).
- `orders` (Каса та Фінанси): Логіка POS-терміналу. Обробка роздрібних та гуртових замовлень, управління транзакціями (передоплата, дооплата, повернення), генерація PDF-чеків, відстеження балансу кас.
- `products` (Номенклатура): Каталог товарів, автоматичний розрахунок роздрібної ціни на основі націнки, завантаження фото в Cloudinary, масовий імпорт з CSV.
- `warehouses` (Склади та Service Desk): - *Склади:* Облік залишків, фіксація історії руху товарів (Stock Movements), внутрішні переміщення.
  - *Ремонти:* Ведення сервісних робіт (FSM - машина станів), валідація унікальності комірок зберігання для активних ремонтів.
- `activity_log` (Аудит): Автоматичне логування всіх CRUD дій адміністраторів та користувачів для аудиту безпеки.

## Швидкий старт (За допомогою Docker - Рекомендовано)

Найпростіший спосіб запустити проєкт разом із базою даних PostgreSQL.

### 1. Клонування репозиторію
git clone https://github.com/Kurioon/erp-pos-system
cd kurioon-erp-pos-system


### 2. Налаштування змінних оточення

Створіть файл .env з прикладу:

cp .env.example .env


Відкрийте .env та впишіть свої ключі для Cloudinary (без них не працюватиме завантаження фото).

### 3. Запуск контейнерів

docker-compose up --build


Додаток запуститься за адресою: http://localhost:8000

## Локальний запуск (Без Docker)

Якщо ви хочете запустити проєкт через вбудований сервер Django (буде використано локальну базу SQLite за замовчуванням):

1. Створіть та активуйте віртуальне середовище:
python -m venv venv
source venv/bin/activate  # Для Mac/Linux
# АБО
venv\Scripts\activate     # Для Windows



2. Встановіть залежності:
pip install -r requirements.txt



3. Застосуйте міграції бази даних:
python manage.py migrate



4. Запустіть сервер:
python manage.py runserver




## Заповнення бази тестовими даними (Seeding)

Для швидкого тестування системи ви можете заповнити базу готовими даними (товари, склади, каси, курси валют та користувачі).

Виконайте команду в терміналі (при запущеному віртуальному середовищі або всередині Docker-контейнера):

python manage.py shell < apps/seed.py


Тестові облікові записи:

* Адміністратор: admin@erp.com / Пароль: admin123
* Продавець: seller@erp.com / Пароль: seller123

## Безпека та CORS

Для локальної розробки у файлі settings.py налаштовано дозволи для запитів з Frontend-додатку (за замовчуванням http://localhost:5173).
Якщо ваш Frontend працює на іншому порту, оновіть змінні CORS_ALLOWED_ORIGINS та CSRF_TRUSTED_ORIGINS у файлі .env.

Для генерації PDF-файлів використовується кастомний шрифт DejaVu Sans, який підтримує кирилицю (налаштовано у config/pdf_utils.py).
